# OpenFilm 技术配图规范 `technical-v1`

风格基准：`film-camera-advance-mechanism-zh` 的机械信息图。沿用它的克制、编辑化技术图解语言，但修复其中英文混排、文字越界和局部拥挤问题。

## 固定视觉系统

- 只制作扁平矢量技术信息图，不生成照片感、3D、手绘、水彩、赛博朋克或随机插画风格。
- 背景 `#F7F6F3`，卡片 `#FFFFFF`，主文字 `#1F1F1F`，次文字 `#6B6B6B`，边框 `#D3D1CB` / `#E3E2DE`。
- 强调色只用珊瑚色 `#D97757`、技术蓝 `#3A7CA5`、警示红 `#C04B3A`；其余允许色由校验脚本限定。
- 字体栈：`"Helvetica Neue", Arial, "Noto Sans CJK SC", sans-serif`。标题 38-46px，副标题 18-22px，正文与标注 14-18px，页脚 13-14px。
- 图形使用 1.5-3px 线条、14-16px 卡片圆角和清晰箭头；禁止渐变、阴影、滤镜、纹理、装饰性背景和嵌入位图。
- 画布外边距至少 60px。标题区、主体图区、结论/注释区和页脚必须分区，任何文字不得压线、跨卡片或伸出画布。

## 固定内容结构

- 每篇设计 2-3 个视觉场景，优先 3 个：首图为核心机制/主题总览，后续图为比较、流程、时间轴、结构剖面或数据关系。
- 首图源文件必须为 1200×1200 的 1:1 画布；后续为 1600×900 或 1600×1200。
- 每个场景制作中文和英文两份同构 SVG。构图、图形、颜色、编号和比例保持一致，只翻译文字。
- 中文 SVG 只能出现中文说明；型号、化学式、单位和公认缩写需要保留时，在对应 `<text>` 上设置 `data-technical-token="true"`。
- 英文 SVG 不得出现中文。中英文文章不得共用同一个 R2 图片 URL。
- SVG 根节点必须包含：

```xml
<svg ... data-openfilm-style="technical-v1" data-locale="zh"
  font-family="'Helvetica Neue', Arial, 'Noto Sans CJK SC', sans-serif">
```

- 每个 `<text>` 必须使用明确的 `x`、`y`、`font-size` 和 `data-max-width`。长句必须人工换成多个 `<text>`，不能依赖浏览器自动换行。

## 生成后校验

1. 对中文和英文源文件分别运行 `.pipeline/validate-blog-images.js`。脚本检查风格标记、画布比例、固定色板、语言纯度、文字安全区、宽度与文字间重叠。
2. 校验通过后，逐张打开源 SVG 或渲染预览进行视觉复核，确认：
   - 标题、标签、箭头和数字均完整，无裁切、越界、重叠或乱码；
   - 阅读顺序明确，字号层级一致，卡片间距均匀；
   - 图解与文章事实一致，没有错误部件、错误方向或无意义装饰；
   - 中文图全中文、英文图全英文，两套图构图一一对应。
3. 任一项不通过，只修改失败的 SVG，再重新执行机器校验和视觉复核。未全部通过前禁止上传 R2。
